1 引言
1.1 目的
本说明书用于描述“Cesium仿真平台（模型管理与LOD优化模块）”的用途、运行环境、主要功能、数据结构与对外接口，并提供该模块的源程序材料，以支撑软件著作权登记材料提交与审查。

1.2 范围
本说明书覆盖以下功能范围（以下简称“模型管理与LOD优化模块”）：
(1) 3D Tiles 模型加载与初始化：从 tileset.json 载入模型，设置 LOD 参数并完成首屏视角定位。
(2) 模型要素扫描与列表管理：扫描瓦片要素并生成模型列表，支持模型识别、分类与配置驱动初始化。
(3) 交互选择与属性同步：点击拾取要素并同步到当前选中模型，提供属性查看入口。
(4) 可视性/透明度/配色/高亮：按模型与全局维度调整透明度、显示隐藏、配色与高亮闪烁。
(5) 模型空间变换：按经纬高位置移动模型，按 XYZ 旋转角度旋转模型，支持一键复位。
(6) 地下查看：通过地表透明度与深度检测策略切换地下查看模式。
(7) LOD 优化：提供预设与参数面板，支持应用、撤销、重置；同步展示请求队列、解码处理、内存等运行指标。

不在本说明书范围内的内容包括：地质分析、测量分析、场景仿真交互、实时监控、系统工具等其他模块，以及完整部署运维方案。

1.3 术语、定义与缩略语
Cesium：三维地球/地图渲染引擎，本模块使用其 3D Tiles 与拾取机制管理模型要素。
3D Tiles：Cesium 3D Tileset 规范的数据格式，本模块以 tileset.json 为入口加载。
Tileset：Cesium.Cesium3DTileset 对象，承载瓦片树、要素与 LOD 配置。
Feature：Cesium.Cesium3DTileFeature，对应瓦片要素实例，可设置 color 与属性读取。
LOD：Level of Detail，细节层次；通过屏幕空间误差、缓存预算、动态误差等参数影响加载策略。
SSE：Screen Space Error，屏幕空间误差。

2 软件概述
2.1 软件用途
“Cesium仿真平台”用于在三维场景中展示矿山/工程模型并提供交互与分析能力。本说明书所述“模型管理与LOD优化模块”用于完成模型载入、要素管理与交互控制，并通过可配置的 LOD 策略提升渲染性能与加载体验。

2.2 软件运行环境
2.2.1 软件环境
前端框架：Vue 3.x
构建工具：Vite 5.x
三维引擎：Cesium 1.138.0
状态管理：Pinia 2.x
建议运行：Chrome/Edge 等现代浏览器

2.2.2 硬件环境（建议）
CPU：4 核及以上
内存：8GB 及以上（大型瓦片模型建议 16GB）
显卡：支持 WebGL 2.0 的独立/集成显卡

2.3 主要功能概述
(1) 模型加载与复位：加载 tileset.json，保存原始矩阵与中心点，实现一键复位。
(2) 要素管理：扫描瓦片树并缓存 Feature 映射，支持按 ID/名称等多来源识别。
(3) 交互选择：鼠标点击拾取 Feature 并定位到模型列表项，实现属性同步。
(4) 视觉控制：对模型执行显示/隐藏、透明度与配色更新，提供高亮闪烁。
(5) 变换控制：按位置与旋转参数修改 tileset.modelMatrix，并提供 UI 面板调节。
(6) 地下查看：通过 Globe 透明度与深度检测参数控制地下视角。
(7) LOD 优化：通过预设与滑块配置实现 SSE、缓存预算、动态误差、中心优先、跳级等策略；展示运行统计信息。

2.4 关键技术与创新点（模块层面）
(1) 要素扫描与映射缓存：对 tileset 瓦片树进行遍历并建立 featureMap，提高模型级控制效率。
(2) 配置驱动的模型列表初始化：支持从 feature.json 读取映射并补齐缺失要素，提高可维护性。
(3) 以组合式函数统一封装控制面：将模型加载、拾取、变换与 LOD 更新统一封装为 useModel，便于多个面板复用。
(4) LOD 可视化调参：以预设 + 参数分组的方式提供可解释配置，并实时展示运行态指标辅助定位性能问题。

3 功能说明（模型管理与LOD优化模块）
3.1 功能边界与约束
(1) 本模块以浏览器端为运行环境，受限于 WebGL、显存与网络带宽。
(2) 3D Tiles 的要素可操作性依赖数据是否携带可读属性与是否支持 color 覆盖。
(3) LOD 参数配置仅对当前 tileset 生效，且参数项需为 Cesium3DTileset 的可写属性。
(4) 变换以 tileset.modelMatrix 为入口；若外部对 modelMatrix 做额外变更需协调复位逻辑。

3.2 功能流程
3.2.1 模型加载与初始化
(1) 读取配置文件路径，推导 tileset.json 路径。
(2) 从预设或默认配置组装 LOD 参数，调用 Cesium3DTileset.fromUrl 加载。
(3) 将 tileset 加入场景，绑定 loadProgress/initialTilesLoaded/allTilesLoaded 等事件以更新运行指标。
(4) 保存 originalModelMatrix 与 originalBoundingSphereCenter，用于复位与变换。
(5) 应用默认位置与旋转变换，并将相机重置到模型视角。

3.2.2 要素扫描与模型列表生成
(1) 使用 FeatureManager 扫描 tileset.root 的瓦片树。
(2) 对每个 tile.content 中的 feature 计算 featureId，并存入 featureMap。
(3) 若存在配置文件（modelMappings），按配置初始化模型列表并补齐缺失要素；否则生成默认列表并进行关键字分类。

3.2.3 交互拾取与属性同步
(1) 创建 ScreenSpaceEventHandler 并监听左键点击。
(2) 对点击点执行 scene.pick，若拾取到 tileset 的 Feature 且当前不处于测量模式，则触发选择。
(3) 基于 featureId 匹配模型列表项并更新 selectedModel；若未匹配则生成临时模型并加入列表。

3.2.4 视觉控制
(1) 显示隐藏：通过调整 feature.color alpha 值模拟可见性。
(2) 透明度：综合模型透明度与全局透明度计算最终 alpha。
(3) 配色：将 CSS 颜色解析为 Cesium.Color 并与透明度合成。
(4) 高亮：切换为高亮色并在指定时长后恢复原始颜色。

3.2.5 模型变换与地下查看
(1) 变换：将 tileset.modelMatrix 重置到 originalModelMatrix；按经纬高计算平移矩阵；在 ENU 坐标系下按 XYZ 角度叠加旋转。
(2) 地下查看：保存并调整 scene.globe.translucency 与深度检测相关参数，实现地表半透明与地下视角。

3.2.6 LOD 优化配置与运行统计
(1) LOD 面板提供预设选择与分组参数编辑。
(2) 应用时对输入参数执行 sanitize 与 clamp，更新到 store 并通过属性赋值应用到 tileset。
(3) 运行统计由 tileset.loadProgress 与 ready 状态事件驱动更新，展示请求队列、处理量与内存占用。

3.3 数据结构说明
3.3.1 ModelItem（模型条目）
字段            类型      说明
id              string   模型唯一标识
featureId        string   对应 tileset feature 的标识
name            string   显示名称
visible         boolean  可见状态
opacity         number   透明度（0-100，内部用于换算 alpha）
styleProperties object   配色、可见、透明度等样式属性
geologyProperties object  扩展属性集合

3.3.2 位置与变换
position  { longitude, latitude, height }
transform { rotationX, rotationY, rotationZ }

3.3.3 LOD 配置与运行指标
lodConfig   object  主要参数：maximumScreenSpaceError、cacheBytes、dynamicScreenSpaceError 等
lodRuntime  object  pendingRequests、tilesProcessing、initialTilesLoaded、allTilesLoaded、lastUpdatedMs

3.4 主要接口说明
3.4.1 useModel（src/composables/useModel.js）
initModel(viewer)：初始化模型与事件处理。
onConfigChange(configFilePath)：加载配置并完成 tileset 加载与列表初始化。
toggleModelVisibility(model)：切换模型显示。
updateModelOpacity(model)：更新单模型透明度。
updateGlobalOpacity(value)：更新全局透明度并批量应用。
highlightModel(model)：高亮显示。
updatePosition(position)、updateTransform(transform)：更新位置与旋转。
setUndergroundViewEnabled(enabled)、updateGlobeTranslucency(params)、enterUndergroundView()：地下查看相关。
updateLodConfig(config)、resetLodConfig()：更新/重置 LOD 配置。
getLodStats()：读取 tileset 的运行统计信息。
startGlobalFpsMonitoring()：FPS 监控。

3.4.2 FeatureManager（src/features/model/featureManager.js）
scanAndStoreFeatures()：遍历瓦片树并缓存 featureMap。
getFeatureId(feature)：从多来源属性解析要素 ID。
toggleModelVisibility(model)：基于 alpha 模拟显示隐藏。
updateModelOpacity(model, globalOpacity)：按模型与全局透明度合成。
updateModelColor(model, hexColor, globalOpacity)：设置模型配色。
highlightModel(model)：高亮并恢复。

3.4.3 模型变换工具（src/features/model/transform/modelTransform.js）
moveModelToPosition(tileset, lon, lat, height, originalCenter)：平移模型。
rotate(tileset, rx, ry, rz, center)：在 ENU 坐标系下旋转模型。
applyModelTransform(tileset, originalModelMatrix, position, transform, originalCenter)：应用组合变换。

3.4.4 状态仓库与配置（src/utils/stores/modelStore.js，src/config/modelConfig.json）
modelStore：保存模型列表、当前 tileset、LOD 配置、运行统计、地下查看状态等。
modelConfig.json：默认 LOD 参数、预设、默认位置与默认旋转。

3.4.5 LOD 面板（src/features/optimization/LodPanel.vue）
预设：高质量/平衡/高性能/自定义。
参数编辑：按分组提供滑块与输入框，并对输入进行范围约束。
操作：应用、撤销、重置。

3.5 异常与容错处理
(1) Viewer 未初始化：load3DModel 直接返回错误提示。
(2) tileset 重新加载：移除旧 tileset 与事件监听，重置运行指标。
(3) 属性赋值失败：对 tileset 属性设置进行 try/catch，避免破坏渲染。
(4) 高亮恢复异常：恢复逻辑包含 try/catch，避免影响主流程。

4 操作说明（面向使用者）
4.1 启动与进入模块
(1) 启动前端应用并进入主界面。
(2) 打开“模型控制/LOD优化”等功能面板。

4.2 模型加载与要素管理
(1) 选择并加载配置文件（feature.json），系统将推导 tileset.json 并加载模型。
(2) 加载成功后，模型列表会自动生成或按配置初始化。
(3) 在场景中点击模型区域可完成拾取选择并同步属性。
(4) 可对模型执行显示隐藏、透明度调整、配色与高亮。

4.3 模型变换与地下查看
(1) 在“模型控制”面板中调节经纬高与 XYZ 旋转角度，变换将实时应用。
(2) 开启地下查看并调整地表/背面不透明度，点击“进入地下视角”进入地下观察。
(3) 点击“重置视角/重置模型”恢复默认状态。

4.4 LOD优化配置
(1) 选择预设（高质量/平衡/高性能）并自动应用。
(2) 在参数分组内按需调节 SSE、缓存、动态误差、中心优先、跳级等参数。
(3) 点击“应用到场景”生效；点击“撤销/默认”回退。
(4) 观察运行状态中的请求队列、处理量、内存与 FPS，辅助定位性能瓶颈。

054     Cesium.Matrix4.multiply(rotateZMatrix, rotateMatrix, rotateMatrix)
055   }
